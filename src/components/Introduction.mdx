## Component Categories

### Core Chat Components

- **ChatContainer**: The main message container with auto-scrolling
```tsx
<ChatContainer messages={messages} maxHeight="600px" />
```

- **ChatBubble**: Individual message bubbles for user and AI responses
```tsx
<ChatBubble 
  message="Hello!"
  isUser={true}
  timestamp="12:30 PM"
/>
```

- **ChatInput**: Text input with file attachment support
```tsx
<ChatInput
  onSubmit={handleSubmit}
  afterInput={<FileSelector />}
/>
```

### Control Components

- **ChatSelector**: Switch between different chat threads
- **ModelSelector**: Choose between different AI models
- **ReasoningToggle**: Toggle AI reasoning/explanation mode

### File Management

- **FileSelector**: File upload with validation
- **SelectedFiles**: Display and manage uploaded files

### Feedback Components

- **AIProcessingIndicator**: Show AI processing state with progress

## Theming

The library uses a powerful theme provider that makes it easy to customize every aspect:

```tsx
<ThemeProvider initialTheme="default"> // or "fruit", "skylight", etc.
  <ChatContainer>
    {/* Your chat content */}
  </ChatContainer>
</ThemeProvider>
```

### Available Themes

- **Default**: Clean, modern design
- **Fruit**: Apple-inspired messaging interface
- **Skylight**: Microsoft Fluent Design System
- **Forest**: Nature-inspired green theme
- **Twilight**: Elegant purple theme

### Custom Themes

Create your own theme by extending the base theme:

```tsx
const customTheme = {
  colors: {
    primary: '#2E5CD3',
    background: '#FFFFFF',
    userBubbleBackground: '#2E5CD3',
    // ... other color tokens
  },
  spacing: {
    xs: '4px',
    sm: '8px',
    // ... other spacing tokens
  }
  // ... other theme tokens
};

<ThemeProvider initialTheme={customTheme}>
  {/* Your app */}
</ThemeProvider>
```

## Architecture

The library is built with a modular architecture where each component:
- Is fully self-contained
- Has consistent prop interfaces
- Uses shared theme tokens
- Supports composition
- Includes comprehensive tests

## Best Practices

1. **Theme Usage**
   - Always wrap your app with ThemeProvider
   - Use theme tokens instead of hard-coded values
   - Consider light/dark mode support

2. **Component Composition**
   - Use ChatContainer as the main wrapper
   - Combine with ChatInput for a complete interface
   - Add file handling as needed

3. **Accessibility**
   - All components support keyboard navigation
   - ARIA labels are included
   - Color contrast meets WCAG guidelines

4. **Performance**
   - Messages are virtualized for large chats
   - Optimized re-renders
   - Lazy loading for file previews
